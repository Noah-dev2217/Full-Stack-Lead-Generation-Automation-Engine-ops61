## Conventions

- **Status enums are lowercase, snake_case.** Don't deviate; multiple pipelines and the operator's Discord notifications depend on exact string matches.
- **Timestamps are UTC ISO 8601** (`2026-06-27T15:30:00Z`). Sheets displays in operator's local TZ.
- **No edits to source-of-truth columns by operator.** Operators may edit `status` to move rows through the funnel, but never edit `Personalized_First_Line`, `Research_Summary`, `video_drive_link`, etc. If they need a regeneration, they flip status back to `pending` and the pipeline re-processes.
- **`Personalized_First_Line` = `[NO_CONTEXT]`** is the sentinel for "Perplexity returned nothing usable." Operator filters these out at send time. Pipeline never errors on no-context; it always writes a row.
- **All four tabs are independent.** No foreign keys, no joins. Each row is self-contained. Cross-pipeline aggregation (if ever needed) happens in a separate analytics layer, not in these tabs.

## Scale notes

At 100 Loomless leads/day = 3,000 rows/month per tab. Sheets handles this fine (Sheets cap is 10M cells / ~5M rows). Threshold for migration to Supabase: ~400 leads/day sustained = 12K rows/month. Document the migration trigger in the plan; don't preempt it.
